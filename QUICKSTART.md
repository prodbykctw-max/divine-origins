# Quickstart

Three things you can do with this repo, in increasing depth. For the full picture
read **[ASSESSMENT.md](./ASSESSMENT.md)**; for orientation read
**[CLAUDE.md](./CLAUDE.md)**.

## 1. Read the spec

- **[SOURCE-MAP-SPEC-COMPLETE.md](./SOURCE-MAP-SPEC-COMPLETE.md)** — every spec doc + scripts + test record, in one file (also `.pdf`).
- Or read them individually under `docs/`, in order `00` → `06`.

## 2. Run the app

The runnable application is in **`app/vite/`** — a React + Vite build of
"Parallels of the Gods" (four-tier canvas, detail panel, scholarly mode, search,
tradition filters, JSON upload).

```bash
cd app/vite
npm install
npm run dev
```

It embeds the v0.1.0 seed inline, but the upload button accepts any dataset
matching the schema — so you can drop in `data/source_map_seed_data_v080.json`
(the clean canonical seed) directly.

Prefer no build step? Open a standalone HTML build in a browser:
- `app/parallels-of-the-gods-v050.html`
- `app/divine-origins-library-v050.html` (the larger "Divine Origins" library)

The v0.5.0 HTML is generated by the Python pipeline in `app/build-v050/`.

## 3. Work with the data

All seed generations live in `data/` (v0.1.0 → v0.8.0). **v0.8.0 is canonical.**

```bash
# validate any seed (exits non-zero on a Severity-1 defect)
python3 tools/validate_seed_data.py data/source_map_seed_data_v080.json

# regenerate the clean v0.8.0 from v0.6.5 structure + v0.7.0 citations
python3 tools/forward_port.py

# validate the documentation (102/102)
python3 scripts/self_test.py
```

Per-version defect reports are in `reports/`; the audit and forward-port writeups
are `reports/AUDIT_v070.md` and `reports/FORWARD_PORT_v080.md`.

## CI

`.github/workflows/ci.yml` runs the docs self-test and validates the canonical
v0.8.0 seed on every push and PR.
