## What this repo is

**The Source Map / Parallels of the Gods** — a comparative-cosmology project that
maps every documented deity, divine concept, and cosmological figure across world
traditions onto a four-tier Gnostic framework (Source / Council / Demiurge /
Archons + Cross-Tier). It is now both a **specification** and a **working data +
app** project.

## Two test harnesses (don't confuse them)

- **`scripts/self_test.py`** validates the **documentation** (terms, JSON blocks,
  cross-refs). Bar: **102/102**. Run after any doc edit before committing.
- **`tools/validate_seed_data.py`** validates the **seed data** (foreign keys,
  dangling refs, self/mislinked parallels, triage contradictions). Exits non-zero
  on any Severity-1 defect. CI gates the **canonical v0.8.0** file with it.

A green self-test says nothing about the data. Always run the data validator on
any seed you touch.

## When the user asks you to do something

### "Validate / fix the data"
Run `python3 tools/validate_seed_data.py data/source_map_seed_data_v080.json`.
The canonical seed must stay at 0 Severity-1 / 0 Severity-2. To regenerate it
from sources, `python3 tools/forward_port.py`. Never hand-edit in a way that
reintroduces dangling `parallel_facets` or orphaned `tradition_id` FKs.

### "Expand the node list" / "add a tradition"
Apply the seven-step methodology in `docs/02` to every node, with all schema
fields from `docs/01` §3 + the additions in `docs/02` step 6. Write results into
a new `data/source_map_seed_data_vNN.json`, then **validate it** and commit a
manifest. Extend the parallel table in `docs/01` §5.

### "Run / build the app"
The runnable app is `app/vite/` (`npm install && npm run dev`). It embeds the
seed inline and supports JSON upload, so it can load any
`data/source_map_seed_data_*.json` — including the clean v0.8.0. The standalone
HTML builds in `app/` open directly in a browser. The v0.5.0 HTML is generated by
the pipeline in `app/build-v050/`.

### "Edit a doc"
After any edit, **run `python3 scripts/self_test.py`** before committing. The bar
is **102/102**. If a test fails, the test is usually right and the doc is wrong.

### "Rebuild the PDFs"
`bash scripts/build_pdfs.sh` (needs pandoc + weasyprint). Outputs to `pdfs/`.

## Editorial principles (binding — don't drift)

1. **Proponent voice by default.** Each tradition speaks in its own voice. No disclaimers in default-mode content.
2. **Scholarly mode is a separate panel.** Caveats go in `docs/03`'s panel structure — never interleaved with proponent text.
3. **Structural function over surface similarity.** Two figures match because they perform the same cosmic role, not because they share a name or animal.
4. **Genealogical vs. typological distinguished.** Documented historical transmission is one thing; pattern-based parallel is another. Tag accordingly per `docs/06`.
5. **Living traditions respected.** Practitioners may disagree with comparative readings; the standing caveats in `docs/03` make this explicit.
6. **No invented citations.** Use only the scholars and works listed in `docs/03`. If a citation is needed and not in the library, say so and ask.
7. **Restricted practices: named, not instructed.** `docs/04` describes practice families and risks but does not provide working instructions for traditions that require initiation.

## What you should never do

- Force a Cross-Tier figure into a single tier just to simplify the database
- Match figures by name, iconography, or domain alone
- Add disclaimers in default-mode essence text
- Cite scholars vaguely or invent sources
- Treat the four-tier framework as the only valid lens
- Conflate Sitchin's Anunnaki-as-ETs reading with the structural Enki-as-craftsman reading
- Encourage unguided practice in traditions that explicitly warn against it
- Skip the self-test before committing doc changes
- Commit a seed that fails `tools/validate_seed_data.py` as the canonical file
